RAG Pipeline Architecture

How this RAG pipeline works end-to-end — methods, data flow, and components

🏗️ High-Level Architecture

The pipeline has a single entry point (main.py) that parses CLI args, validates environment variables, and dispatches to the RAGPipeline orchestrator class.

GitHub Provider

  • Single GITHUB_TOKEN
  • Embeddings: text-embedding-3-small via GitHub Models
  • Generation: gpt-4o via GitHub Models
  • Endpoint: models.inference.ai.azure.com

Default Provider

  • OPENAI_API_KEY + ANTHROPIC_API_KEY
  • Embeddings: text-embedding-3-small via OpenAI
  • Generation: claude-sonnet-4-6 via Anthropic
  • Direct API calls

Both providers share the same ChromaDB persistent vector store at ./chroma_db with cosine similarity.

📥 Flow 1: Ingest Documents

python main.py --provider github ingest docs/
1

File Discovery

src/pipeline.py → RAGPipeline.ingest(path)
Recursively scans directory for .pdf, .md, .txt, .markdown files

↓
2

Document Loading

src/ingestion.py → load_pdf() or load_text_file()
PDFs: extracts text per page via pypdf · Text files: reads as single page

↓
3

Chunking

src/ingestion.py → chunk_text(text, size=1000, overlap=200)
Sliding window with smart boundary detection (prefers \n or ". " breaks)

↓
4

Embedding

src/ingestion.py → embed_texts(chunks, client)
Calls text-embedding-3-small API in batches of 100 chunks

↓
5

Store in Vector DB

src/ingestion.py → collection.upsert()
ChromaDB upsert in batches of 500 · ID = MD5(source:page:chunk_index) · Metadata: {source, page, chunk_index}

🔍 Flow 2: Query

python main.py --provider github query "What is harness engineering?"
1

Entry

src/pipeline.py → RAGPipeline.query(question, top_k=5)

↓
2

Embed Query

src/retrieval.py → openai_client.embeddings.create()
Converts the question into a vector using text-embedding-3-small

↓
3

Vector Search

src/retrieval.py → collection.query(query_embeddings, n_results=top_k)
ChromaDB cosine similarity search · Returns top-K hits with text, metadata, and scores

↓
4

Build Context

src/generation.py → _build_context(hits)
Formats chunks as: [1] source (page N)\ntext...\n---\n[2] ...

↓
5

LLM Generation

src/generation.py → generate_answer_compat() or generate_answer()
System prompt: "Answer based strictly on provided context. Cite sources."
GitHub → GPT-4o · Default → Claude Sonnet

↓
6

Return Result

Returns {"answer": "...", "sources": [hits]}

💬 Flow 3: Interactive Chat

python main.py --provider github chat

Same as Flow 2 (Query) but wrapped in a while True loop:

  1. Read user input from stdin
  2. Call pipeline.query(question)
  3. Print the answer
  4. Repeat until user types quit, exit, or q
